# Excel Console MVP

## Goal
- Replace the local web UI as the default internal monitoring surface with an Excel desktop console.
- Keep Python as the only source of truth for data, backtests, validation, experiment records, and tracked memory.
- Keep the local web surfaces frozen until the Excel MVP covers the required internal scenarios and the user explicitly approves deletion.

## Scope
- Workbook path: `artifacts/projects/<project>/excel/ResearchConsole.xlsm`
- Feed path: `artifacts/projects/<project>/excel/feed/`
- Action launchers: `artifacts/projects/<project>/excel/actions/`

The MVP covers:
- current project summary
- strategy board snapshot
- experiment ledger snapshot
- recent run snapshot
- artifact links
- denser dashboard cards for mainline, blocker, next action, and recent validation
- strategy comparison charts
- recent experiment result chart
- embedded preview images for the latest F1 / F2 / R1 comparison plots when those PNG artifacts exist
- safe launcher links for:
  - `excel_export`
  - `research_audit`
  - `agent_cycle --dry-run`
  - `f1_verify`
  - `f2_verify`
  - opening artifacts and tracked memory directories

The MVP does not cover:
- `f2_train`
- configuration editing
- deleting experiments
- direct tracked-memory writes from Excel
- replacing research logic with VBA

## Workbook Mode
- The current machine can open Excel through COM, but it does not expose a writable `VBProject`.
- Because of that, this MVP uses launcher links instead of embedded VBA macros.
- The workbook is generated by saving a staging `.xlsx` file and converting it through Excel `SaveAs(..., 52)` so the final `ResearchConsole.xlsm` is a real Excel-openable macro-enabled package.
- If the primary workbook is still open and locked, export writes a timestamped fallback workbook instead of silently failing.
- A VBA template is still exported under `artifacts/projects/<project>/excel/vba/ResearchConsoleModule.bas` so the macro path can be enabled later if Excel Trust Center settings are changed.

## Freeze Rule
- `apps/web` is in feature freeze.
- `dashboard/app.py` is in feature freeze.
- Delete these web surfaces only after:
  - the Excel console covers overview, experiment ledger, latest verification results, and the safe command set
  - the user confirms the web entrypoints are no longer needed
